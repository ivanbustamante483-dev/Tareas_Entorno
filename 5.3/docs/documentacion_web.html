🛰️ Comparación de Optimización y Documentación en Python

Documentación del proyecto — Actividad de clase DAW

🎯 Objetivo del Proyecto

Crear una aplicación en Python con interfaz gráfica (Tkinter) que compare dos versiones del mismo problema: una versión no optimizada y una versión optimizada.

La aplicación obtiene datos en tiempo real de la API pública Open Notify (posición de la ISS y astronautas en el espacio) y los procesa de dos formas diferentes para demostrar el impacto de la optimización en el rendimiento.

⚠️ NO OPT Versión No Optimizada

Esta versión realiza el procesamiento de datos de forma intencionadamente ineficiente para ilustrar malas prácticas de programación:

  • Bucles innecesarios: Se repiten cálculos dentro de bucles que solo iteran una vez o que recalculan el mismo valor múltiples veces.
  • Búsqueda manual de duplicados: En lugar de usar set(), se recorren listas anidadas para encontrar elementos únicos.
  • Conteo manual de caracteres: Se recorren cadenas carácter a carácter en lugar de usar len().
  • Cálculo manual de estadísticas: En lugar de usar sum(), min(), max(), se implementan bucles manuales.
  • Concatenación de cadenas con +: Se construye el resumen concatenando cadenas una a una, lo cual es más lento.

Ejemplo de código no optimizado

# Calcular suma manualmente
suma_longitudes = 0
for lon in longitudes:
    suma_longitudes = suma_longitudes + lon

# Buscar naves únicas con bucle anidado
naves_unicas = []
for astro in lista_astronautas:
    nave = astro["nave"]
    encontrada = False
    for n in naves_unicas:
        if n == nave:
            encontrada = True
    if not encontrada:
        naves_unicas.append(nave)

✅ OPT Versión Optimizada

Esta versión realiza exactamente el mismo trabajo, pero de forma eficiente:

  • Operadores ternarios: Decisiones simples en una línea.
  • List comprehensions: Extracción de datos en una sola expresión.
  • Funciones integradas: Uso de sum(), min(), max(), len(), set().
  • f-strings: Construcción de cadenas eficiente y legible.
  • math.hypot(): Cálculo directo de la distancia.

Ejemplo de código optimizado

# Naves únicas con set
naves_unicas = list({a["nave"] for a in lista})

# Estadísticas con funciones integradas
longitudes = [len(n) for n in nombres]
media = sum(longitudes) / len(longitudes) if longitudes else 0

# Nombre más largo con max() y key
nombre_mas_largo = max(nombres, key=len) if nombres else ""

📈 cProfile Profiling con cProfile

Cada panel de la aplicación muestra en su parte inferior las estadísticas de rendimiento generadas por el módulo cProfile de la biblioteca estándar de Python.

¿Qué es cProfile?

cProfile es un perfilador (profiler) determinístico incluido en Python. Registra cada llamada a función realizada durante la ejecución, midiendo:

  • ncalls: Número de veces que se llamó a la función.
  • tottime: Tiempo total pasado dentro de la función (sin contar subfunciones).
  • percall: Tiempo medio por llamada.
  • cumtime: Tiempo acumulado incluyendo subfunciones.

¿Cómo se usa en este proyecto?

profiler = cProfile.Profile()
profiler.enable()
resultado = funcion(*args)
profiler.disable()

stats = pstats.Stats(profiler)
stats.sort_stats("cumulative")
stats.print_stats(10)

Esto permite ver en tiempo real qué funciones consumen más recursos y comparar directamente ambas versiones.

❓ Botón Help y Docstrings

Cada panel de la aplicación incluye un botón Help que, al pulsarse, abre una ventana emergente con la documentación de todas las funciones del módulo utils.py.

¿Qué son los docstrings?

Los docstrings son cadenas de documentación que se colocan justo debajo de la definición de una función, clase o módulo en Python. Sirven para describir:

  • Qué hace la función.
  • Qué parámetros acepta.
  • Qué valor retorna.
def obtener_posicion_iss():
    """
    Obtiene la posición actual de la Estación Espacial Internacional.

    Parámetros:
        Ninguno.

    Retorna:
        dict: Diccionario con latitud, longitud, timestamp, exito, error.
    """

La ventana Help usa el módulo inspect para leer estos docstrings en tiempo de ejecución y mostrarlos formateados.

🚀 Instrucciones de Ejecución

1. Instalar dependencias

pip install -r requirements.txt

2. Ejecutar la aplicación

python main.py

3. Generar PDF de documentación

python docs/generar_pdf.py

Nota: Se requiere conexión a Internet para que la API funcione. La aplicación maneja errores de conexión mostrando un mensaje en los paneles.
